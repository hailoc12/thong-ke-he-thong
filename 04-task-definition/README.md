# Task Definition

Clear problem statements and success criteria.

## How to Use
- Start every project by creating a task definition here
- Define what success looks like
- Set constraints and scope boundaries

## Note Format
```markdown
# Task: [Task Name]

## Problem Statement
[What problem are we solving?]

## Success Criteria
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Scope
**In Scope**:
- Item 1
- Item 2

**Out of Scope**:
- Item 1
- Item 2

## Constraints
- Time: [deadline if any]
- Resources: [limitations]
- Technical: [technical constraints]

## Stakeholders
- [Who cares about this?]
```
