# Quick Solutions

Fast, pragmatic implementations.

## When to Use
- Time pressure requires speed over perfection
- Problem is well-understood
- Solution is temporary or experimental
- Low stakes (easily reversible)

## How to Use
- Document the solution and its limitations
- Note what would make it a quality solution
- Tag for potential future improvement

## Note Format
```markdown
# Quick Solution: [Solution Name]

**Date**: [Date]
**Problem**: [[link to task-definition]]
**Time Spent**: [how long]

## Solution
[Description of the solution]

## Implementation
[Code, steps, or details]

## Limitations
- Limitation 1
- Limitation 2

## To Make This Quality
[What would need to change for this to be production-ready]

## Status
Working | Needs Improvement | Deprecated
```
