# Follow Up

Next actions and ongoing work.

## How to Use
- Track items that need follow-up after project completion
- Document future improvements and enhancements
- Keep track of dependencies on external factors

## Note Format
```markdown
# Follow Up: [Item Title]

**Priority**: High | Medium | Low
**Due**: [Date if applicable]
**Depends On**: [External dependency if any]

## Description
[What needs to be followed up on?]

## Context
[Why is this important?]

## Action Items
- [ ] Action 1
- [ ] Action 2

## Status
Pending | In Progress | Blocked | Done

## Notes
[Any relevant context]
```

## Categories
- **Improvements**: Things to make better later
- **Dependencies**: Waiting on external factors
- **Reviews**: Items needing periodic review
- **Handoffs**: Items to pass to others
